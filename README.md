# Book Contents Formatter Package

This README is for developers and maintainers working on the repository itself: architecture, code areas, data flow, and the current technical documentation set.

If you want the operational guide for running the tools as an end user, read [README_USER.md](README_USER.md).

## Developer Scope

The repository spans four main areas:

1. Word front matter and document assembly.
2. Index extraction, correction, curation, and rendering.
3. EPUB generation, styling, and verification.
4. Config-driven task orchestration and data handoff.

Most Word-based workflows are Windows-only because they depend on Microsoft Word COM automation.

## Source Layout

Key code areas:

- `src/front_matter/` front matter builders and related support code.
- `src/index/` index generation, correction, merge, and verification.
- `src/final_book/` final Word assembly.
- `src/ebook/` EPUB builder, styling, and verification.
- `src/utils/` shared utility scripts.

Key data areas:

- `books/book_project.json` selects the active book.
- `books/<book_slug>/inputs/` holds source manuscripts and assets for one book.
- `books/<book_slug>/work/` holds intermediate JSON, reports, and temp data.
- `books/<book_slug>/outputs/` holds generated stage outputs.
- `books/<book_slug>/release/` holds approved snapshots for that book.

## Configuration Model

The repository prefers JSON configuration over hardcoded paths.

Important config files include:

- `src/front_matter/builders/front_matter_builder.config.json`
- `src/ebook/ebook_builder.config.json`
- `src/index/build/index_builder.config.json`
- `src/final_book/full_book_assembler.config.json`

Paths are generally relative to project root, and downstream tasks consume metadata generated by previous steps when available.
